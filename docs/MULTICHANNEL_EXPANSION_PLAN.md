# Multichannel Expansion Plan — Kukmincalc

> 블로그/가이드 → 스레드 → 지식인 → 유튜브 → 이미지 → 앱 으로 확장.  
> Disney (사용자 여정) + Bezos (재방문) + Aristoteles (확장 가능 구조).

---

## 0. 확장 체인 (한 번 만든 자산 → 6개 채널 재활용)

```
가이드 페이지 1편 (마스터 콘텐츠)
  ↓
[1] 네이버 블로그 요약 글 + 백링크
[2] 네이버 지식인 답변 (관련 질문 검색)
[3] 스레드 / 인스타 카드 뉴스
[4] 유튜브 쇼츠 (30~60초)
[5] 유튜브 롱폼 (5~10분, 가이드 그대로 영상화)
[6] 이미지 인포그래픽 (1200×630 OG / 1080×1080 SNS)
```

각 채널의 결과는 다시 가이드 페이지로 유입 + 계산기 사용 + 공식 링크 클릭으로 수렴.

## 1. 채널별 변환 규칙

### 1-1. 가이드 페이지 → 네이버 블로그 요약

- 본문은 **80% 이내** 분량으로 축약.
- 자체 인사이트 1~2 문단 추가 (단순 복붙 X).
- 원본 캐노니컬 링크 1개.
- 발행 후 1~3일 내 송출.

### 1-2. 가이드 페이지 → 네이버 지식인 답변

- 가이드 페이지의 "1분 계산 예시" 그대로 복사.
- 결론(숫자) → 산식 → 출처 → 계산기 링크 순.
- 답변마다 다른 표현으로 변형 (도배 방지).

### 1-3. 가이드 페이지 → 스레드 카드

- 1~3 슬라이드.
- 1슬라이드: 결론 1줄.
- 2슬라이드: 산식 / 숫자 1개.
- 3슬라이드: "더 정확한 계산은 kukmincalc.com" (광고 표현 X).

### 1-4. 가이드 페이지 → 유튜브 쇼츠 대본

```
[0~3초] 결론 1줄 (강조 자막)
[4~15초] 산식 + 화면 캡처
[16~40초] 1분 계산 예시
[41~55초] 정책 변경 안내 (해당 시)
[56~60초] kukmincalc.com 노출
```

### 1-5. 가이드 페이지 → 유튜브 롱폼 대본

가이드 H2 구조 그대로 = 영상 챕터.  
챕터당 1~2분, 총 5~10분.

### 1-6. 가이드 페이지 → 인포그래픽

OG 이미지 응용 + 시즌별 컬러 바리에이션.

## 3. 성장 우선순위 (Sun Tzu)

`docs/SEO_CONTENT_PLAN.md` 의 P0 → 콘텐츠 발행 → 외부유입 → 측정 → P1 / P2 재정렬 → 가이드 페이지 시리즈 누적 → PWA/앱화.

다음 단계로 가기 전 충분히 발행한 가이드 수 기준:
- 가이드 12편 발행 → 외부유입 루틴 안정화 → 결과 공유 / 토스식 카드 등으로 확장.
- 가이드 30편 + 사용자 행동 데이터 → 질문형 플로우 + 체크리스트 + 시즌 알림.

너무 일찍 기능 늘리지 않는다 (Drucker: "하지 말아야 할 일").

## 6. GPTS/Gems 운영 체인 매핑 (Kukmincalc 선별)

> 첨부한 애드센스팜 GPTS/Gems 운영 가이드는 **계산기·도구형 사이트에 직접 맞는 것만** 선별 반영. 일반 블로그·쇼핑·여행·실버채널·감성 유튜브 류는 의도적으로 제외.

### 6-1. 즉시 반영 (Phase 3-1 문서화)
| GPT/Gem | Kukmincalc 적용 | 반영 위치 |
|---|---|---|
| 황금 키워드 추출 비서 | 키워드 클러스터 + 제목 후보 | `SEO_CONTENT_PLAN.md` |
| 구글/네이버 SEO 글쓰기 비서 | title/meta/H1/FAQ 슬롯 | `SEO_CONTENT_PLAN.md` + `CONTENT_PRODUCTION_PIPELINE.md` + `index.html` |
| 지식인 답변 비서 | 질문형 답변 템플릿 | `EXTERNAL_TRAFFIC_PLAYBOOK.md` §5-1 |
| 스레드/네이버 요약/커뮤니티 봇 | 외부유입 루틴 (도배 금지) | `EXTERNAL_TRAFFIC_PLAYBOOK.md` §5-2~5-4 |
| 이미지/유튜브 확장 | 구현 X, 후속 확장으로만 정리 | `MULTICHANNEL_EXPANSION_PLAN.md` §1 변환 규칙 |

### 6-2. 후순위 (Kukmincalc-fit 조건이 갖춰질 때 부분 참조)
- 쿠팡/쇼핑전환 글쓰기 비서 — 대출비교 / 자동차보험 / 세무신고 같은 Kukmincalc-fit 제휴가 실제 등장하면 그때만 참조.
- 유튜브 쇼츠/롱폼 대본 봇 — 가이드 30편 이상 누적 후 (§3 "성장 우선순위" 참조).
- 이미지/카드뉴스 봇 — 외부유입 루틴이 안정화되고 쇼츠/SNS 채널을 운영할 때만.

### 6-3. 제외 대상 (Kukmincalc 에 해당 안 됨)
다음은 일반 블로그·쇼핑·취미 채널용이므로 **억지로 끼워 넣지 않는다**:

```
네이버 애드포스트 승인비서        ← Kukmincalc는 AdSense 운영, 애드포스트 X
여행블로그 수익화 비서             ← 카테고리 X
쿠팡 쇼핑전환 글쓰기 비서          ← 쇼핑 카테고리 X (단, 6-2 의 부분 참조 예외)
쇼핑 스크립트 작가                 ← 쇼핑 카테고리 X
실버채널 GPTS                      ← 타깃 X
감성 유튜브 대본 템플릿            ← 톤 불일치 (계산기 사이트는 결론형·숫자 우선)
일반 블로그 글쓰기 전체 루틴       ← 가이드는 12단계 표준 공정만 사용 (`CONTENT_PRODUCTION_PIPELINE.md`)
사진 프롬프트 예시 전체            ← 채널 X
다음·네이버 일반 블로그 플랫폼 구조 ← 정적 사이트 구조에 부적합
```

**원칙:** Kukmincalc 는 일반 블로그가 아니라 계산기·도구형 사이트다. GPTS/Gems 자료는 "운영 체인" 만 추리고, 톤·플랫폼·카테고리가 안 맞는 것은 반영하지 않는다.
